# Alt3rSnap v0.2 M4 — Snap (Headline Feature) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Milestone 4 of v0.2 — Alt+Left-drag near a monitor edge/corner shows a translucent ghost preview of the target snap zone; release commits the snap; Space suspends; zones are configurable per-user; multi-monitor + DPI-aware; move-only (resize never snaps). Deliverable: `main` branch with all tests green, the v0.2 `TESTING.md` checklist ready, and a manually-verified Windows binary ready for the `v0.2.0` tag.

**Architecture:** Strictly additive to the v0.1+M1..M3 engine/adapter split. Engine gains a new pure module `engine::snap` plus four actions (`ShowSnapPreview`, `HideSnapPreview`, `ApplySnapRect`, and the event `DragAborted`). `Moving` state gains an `Option<SnapSession>` with drag-local baked zones — immutable for the lifetime of the drag. Adapter gains two new Windows-only modules (`overlay.rs`, `monitors.rs`), a `WinApi` trait + `RecordingWinApi` mock for Linux-runnable ordering tests, and converts `tool_window.rs` from message-only to hidden top-level so broadcast messages (`WM_DISPLAYCHANGE`, `WM_SETTINGCHANGE`) are received. Config gains a `[snap]` section, a tolerant-string-at-file-layer pattern reused from M1/M2, and a `RuntimeConfig { engine, adapter }` split so overlay-paint knobs (`preview_opacity`) never leak into engine code.

**Tech Stack:** Rust 2021, existing `windows` v0.58 features (adds `Win32_Graphics_Gdi` already present + nothing new). No new crates. `smallvec` is introduced (small-sized snap-zone vector, up to 11 entries). `RecordingWinApi` uses std primitives only. All new automated tests run on Linux CI.

**Milestone reference:** Spec `docs/superpowers/specs/2026-04-21-alt3rsnap-v0.2-design.md` §1, §2, §3.1–§3.6, §4.1–§4.7, §5.1, §5.2, §5.6, §5.8, §6.4, §7.1–§7.3.

## Design invariants (locked during M4 planning)

These are non-negotiable across the plan. Any task that violates them is wrong — prefer re-opening the plan over shipping the violation.

1. **`SnapContext` is immutable for drag lifetime.** Holds monitor snapshot, threshold distances, baked zone rects, restore-guard anchor, and space-suspended-at-start flag. `ReloadConfig` mid-drag affects the *next* drag only. A display topology change mid-drag updates the adapter's cache but never mutates the in-flight `SnapContext`.
2. **`Engine::exit_moving(reason)` is the sole teardown path from `Moving`.** Owns: clearing `MovingState`, clearing `SnapSession.engaged`, emitting `HideSnapPreview` iff engaged, resetting restore-guard state, and the reason-specific cleanup. Every exit path (`LeftUp`, `DragAborted`, `ToggleEnable`, `set_config(enabled=false)`, `FullscreenFocused` resolution) routes through it. Adding a new way to exit `Moving` without going through `exit_moving` is a bug.
3. **Space suspend is explicit.** Pressing Space while engaged → immediately hide preview, clear `engaged`, set `suspended = true`. While suspended, `snap::evaluate` returns `None`. Releasing Space → `suspended = false`; the next `MouseMove` re-evaluates from the current cursor with no stale engaged-zone state.
4. **`DragAborted` scope is narrow.** Emitted only by the adapter on hard failure to continue/apply drag geometry: `TargetInvalid`, `CaptureLost`, `ApplyGeometryFailed`. **Not** emitted for: user cancellation, snap disengagement, restore-guard suppression, or normal `LeftUp`. Keeps the event meaningful and avoids leaking every cleanup reason into one event.

**Open-question defaults (per author review):**

- **Preview fade:** off. Instantaneous show/hide. No animation timer, no fade state, no fade config knob.
- **Maximized-restore guard:** distance-anchored. `SnapSession.restore_guard_active: bool` set true iff the target was maximized at `BeginDrag` and `restore_maximized_on_move=true`. While active, `snap::evaluate` returns `None` until `cursor` has moved ≥ 16 px from `grab`. Once the guard is cleared it stays cleared for the rest of the drag.
- **DPI scaling in snap math:** none. All snap math is physical pixels. `MonitorInfo.scale` is carried in the snapshot for forward-compat but not consumed by v0.2.

## File structure

### Files created

- `src/engine/snap.rs` — zone baking (`bake_zones()`), `snap::evaluate()` decision function, `SnapContext`, `SnapSession`, `EngagedZone`, `SnapZone`, `SnapZoneId`, `MonitorSnapshot`, `MonitorInfo`, `Decision` enum (`Engage`/`Hold`/`Disengage`/`None`). Pure Rust; no adapter imports.
- `src/monitors.rs` — Win32-only. `snapshot()` → `MonitorSnapshot`, plus global `Mutex<MonitorSnapshot>` cache, `request_refresh()`, 200 ms debounce, `mark_dirty_during_drag()` for broadcasts arriving mid-drag.
- `src/overlay.rs` — Win32-only. Lazy-initialised hidden layered top-level window, `show(rect)` / `hide()` API, `WM_PAINT` handler with `FillRect` + `FrameRect` at whole-window alpha via `SetLayeredWindowAttributes(LWA_ALPHA)`.
- `src/win_api_trait.rs` — `trait WinApi` with `set_window_rect`, `show_window`, `is_zoomed`, `capture_mouse`, `release_mouse`, `overlay_show`, `overlay_hide`. `Win32WinApi` impl behind `#[cfg(windows)]`; `RecordingWinApi` impl behind `#[cfg(test)]` (or just `pub` module) for Linux test builds.
- `tests/engine_snap.rs` — full §5.1 table (zone geometry, proximity with hysteresis, zone-switch in single MouseMove, tie-break priority, Space suspension, zone immutability across ReloadConfig, cross-monitor, restore guard).
- `tests/adapter_ordering.rs` — curated `Vec<Action>` fed through `apply_actions()` against a `RecordingWinApi`. Asserts §2.11 (no `set_window_rect` after `apply_snap_rect` within a batch), §2.10 (idempotence), and snap-latch cleanup on `BeginDrag` preprocessing.

### Files modified

- `src/engine/state.rs` — `Event::DragAborted { reason: DragAbortReason }`; `Action::ShowSnapPreview { rect }`, `Action::HideSnapPreview`, `Action::ApplySnapRect { hwnd, rect }`; `DragAbortReason` enum; `BeginDrag` gains `snap: Option<SnapContext>`; `State::Moving` gains `snap_session: Option<SnapSession>`.

### Files intentionally not modified

`src/tray.rs` (tray menu unchanged), `src/fullscreen.rs` (foreground-hook source of FullscreenFocused unchanged), `src/dpi.rs`, `src/elevate.rs`, `src/autostart.rs`, `src/logging.rs`, `src/config_ops.rs` (just continues to call whatever the top-level config loader exposes), `src/swallow_latch.rs`, `src/engine/geometry.rs`, `src/engine/modifiers.rs`, `src/engine/policy.rs`, `src/engine/rules.rs`. No M4 scope there.

**Decomposition rationale:** `engine::snap` is its own module because it's the heart of M4 correctness, it's pure Rust, and it's the piece most likely to grow post-v0.2 (custom zones, closest_edge mode). `overlay.rs` and `monitors.rs` are Win32-specific and each owns a distinct OS resource (a layered popup; a cached `Vec<MonitorInfo>`) — they don't share code. `win_api_trait.rs` is a new file because the `WinApi` trait is the adapter boundary and deserves its own review surface. The `RuntimeConfig` split is not a new file — it lives in `src/config.rs` alongside the bridge that produces it.

  ```rust
  /// Returns the decision for this MouseMove; mutates `engaged` / `last_preview_rect`
  /// appropriately. Never returns Engage while `suspended` or while the restore guard
  /// is active.
  pub fn evaluate(session: &mut SnapSession, cursor: Point) -> Decision {
      if session.suspended {
          if session.engaged.take().is_some() {
              session.last_preview_rect = None;
              return Decision::Disengage;
          }
          return Decision::None;
      }

      if session.ctx.restore_guard_active && !session.restore_guard_cleared {
          let dx = cursor.x - session.ctx.grab.x;
          let dy = cursor.y - session.ctx.grab.y;
          if (dx * dx + dy * dy) < (16 * 16) {
              return Decision::None;
          }
          session.restore_guard_cleared = true;
      }

      let winner = best_candidate(&session.ctx, cursor, session.ctx.engage_px as i32);

      match (&session.engaged, winner) {
          (None, None) => Decision::None,
          (None, Some(z)) => {
              let ez = EngagedZone {
                  id: z.id,
                  target_rect: z.target_rect,
                  entered_at_cursor: cursor,
              };
              session.engaged = Some(ez.clone());
              session.last_preview_rect = Some(z.target_rect);
              Decision::Engage(ez)
          }
          (Some(cur), Some(z)) if cur.id == z.id => Decision::Hold,
          (Some(_), Some(z)) => {
              let ez = EngagedZone {
                  id: z.id,
                  target_rect: z.target_rect,
                  entered_at_cursor: cursor,
              };
              session.engaged = Some(ez.clone());
              session.last_preview_rect = Some(z.target_rect);
              Decision::Engage(ez)
          }
          (Some(_), None) => {
              let dist_px = disengage_distance_to_current(session, cursor);
              if dist_px >= session.ctx.disengage_px as i32 {
                  session.engaged = None;
                  session.last_preview_rect = None;
                  Decision::Disengage
              } else {
                  Decision::Hold
              }
          }
      }
  }

  fn best_candidate<'a>(ctx: &'a SnapContext, cursor: Point, engage_px: i32) -> Option<&'a SnapZone> {
      let mut best: Option<(&SnapZone, u8)> = None;
      for z in &ctx.zones {
          if !cursor_eligible(z, cursor, engage_px, &ctx.monitors.monitors[z.monitor_index]) {
              continue;
          }
          let pr = priority_rank(z.id);
          best = match best {
              None => Some((z, pr)),
              Some((_, br)) if pr < br => Some((z, pr)), // lower rank = higher priority
              Some(x) => Some(x),
          };
      }
      best.map(|(z, _)| z)
  }

  /// True iff the cursor is within `engage_px` of the zone's triggering edge/corner
  /// (edges use `bounds` so auto-hidden taskbar does not defeat the top edge).
  fn cursor_eligible(z: &SnapZone, cursor: Point, engage_px: i32, mon: &MonitorInfo) -> bool {
      use SnapZoneId::*;
      let b = mon.bounds;
      match z.id {
          TopLeftQuarter     => (cursor.x - b.left).abs() <= engage_px && (cursor.y - b.top).abs() <= engage_px,
          TopRightQuarter    => (cursor.x - b.right).abs() <= engage_px && (cursor.y - b.top).abs() <= engage_px,
          BottomLeftQuarter  => (cursor.x - b.left).abs() <= engage_px && (cursor.y - b.bottom).abs() <= engage_px,
          BottomRightQuarter => (cursor.x - b.right).abs() <= engage_px && (cursor.y - b.bottom).abs() <= engage_px,
          TopMaximize        => (cursor.y - b.top).abs() <= engage_px && cursor.x > b.left && cursor.x < b.right,
          BottomMaximize     => (cursor.y - b.bottom).abs() <= engage_px && cursor.x > b.left && cursor.x < b.right,
          LeftHalf           => (cursor.x - b.left).abs() <= engage_px && cursor.y > b.top && cursor.y < b.bottom,
          RightHalf          => (cursor.x - b.right).abs() <= engage_px && cursor.y > b.top && cursor.y < b.bottom,
          LeftThird          => (cursor.x - b.left).abs() <= engage_px,
          MiddleThird        => cursor.x > b.left + (b.right - b.left) / 3
                                && cursor.x < b.left + 2 * (b.right - b.left) / 3,
          RightThird         => (cursor.x - b.right).abs() <= engage_px,
      }
  }

  /// Lower number = higher priority.
  fn priority_rank(id: SnapZoneId) -> u8 {
      use SnapZoneId::*;
      match id {
          TopLeftQuarter | TopRightQuarter | BottomLeftQuarter | BottomRightQuarter => 0,
          TopMaximize => 1,
          BottomMaximize => 2,
          LeftHalf | RightHalf => 3,
          LeftThird | MiddleThird | RightThird => 4,
      }
  }

  /// Distance from the cursor to the nearest edge of the currently engaged zone's
  /// *triggering* edge (used for hysteresis).
  fn disengage_distance_to_current(session: &SnapSession, cursor: Point) -> i32 {
      let Some(ez) = &session.engaged else { return i32::MAX };
      let r = ez.target_rect;
      let dx = 0.max((r.left - cursor.x).max(cursor.x - r.right));
      let dy = 0.max((r.top - cursor.y).max(cursor.y - r.bottom));
      dx.max(dy)
  }
  ```

  Note: `disengage_distance_to_current` uses Chebyshev distance to the engaged zone's rect; it's simple and produces the expected hysteresis behaviour for rectangular zones. If the B3/B4 tests surface a case that needs edge-specific distance (e.g. top-maximize), refactor locally inside `disengage_distance_to_current` without changing the public API.

  ```rust
  #[derive(Debug, Clone, Copy, PartialEq, Eq)]
  pub(crate) enum ExitMovingReason {
      LeftUp,
      DragAborted,
      DisabledFromToggle,
      DisabledFromReload,
  }

  impl Engine {
      /// Central teardown path from `State::Moving`. Emits `HideSnapPreview` iff a
      /// snap_session had `engaged.is_some()`; then emits reason-specific actions.
      /// Caller is responsible for setting the new `self.state` after this returns.
      fn exit_moving(&mut self, reason: ExitMovingReason, actions: &mut Vec<Action>) -> Option<WindowId> {
          let (hwnd, had_engaged, pending_passthrough) = match &self.state {
              State::Moving { hwnd, snap_session, pending_passthrough, .. } => (
                  *hwnd,
                  snap_session.as_ref().and_then(|s| s.engaged.clone()),
                  *pending_passthrough,
              ),
              _ => return None,
          };

          if had_engaged.is_some() {
              actions.push(Action::HideSnapPreview);
          }

          match reason {
              ExitMovingReason::LeftUp => {
                  if let Some(ez) = had_engaged {
                      actions.push(Action::ApplySnapRect { hwnd, rect: ez.target_rect });
                  }
                  actions.push(Action::EndDrag { hwnd });
                  actions.push(Action::CancelMenuActivation);
              }
              ExitMovingReason::DragAborted => {
                  actions.push(Action::EndDrag { hwnd });
              }
              ExitMovingReason::DisabledFromToggle | ExitMovingReason::DisabledFromReload => {
                  actions.push(Action::EndDrag { hwnd });
                  actions.push(Action::CancelMenuActivation);
              }
          }

          // Caller sets state; expose pending_passthrough to caller via a side channel
          // in a follow-up sub-step if needed. For now LeftUp needs it; encode in a
          // small struct return.
          let _ = pending_passthrough; // referenced by LeftUp-specific code below
          Some(hwnd)
      }
  }
  ```

  ```rust
  fn exit_moving(&mut self, reason: ExitMovingReason, actions: &mut Vec<Action>)
  ```

  ```rust
  #![cfg(windows)]

  use alt3rsnap::engine::geometry::Rect;
  use alt3rsnap::engine::snap::{MonitorInfo, MonitorSnapshot};
  use std::sync::{Mutex, OnceLock};
  use std::time::{Duration, Instant};
  use windows::Win32::Foundation::{BOOL, HWND, LPARAM, POINT, RECT};
  use windows::Win32::Graphics::Gdi::{
      EnumDisplayMonitors, GetMonitorInfoW, HMONITOR, MONITORINFO, MONITORINFOEXW,
  };
  use windows::Win32::UI::HiDpi::{GetDpiForMonitor, MDT_EFFECTIVE_DPI};

  static CACHE: OnceLock<Mutex<Cached>> = OnceLock::new();

  struct Cached {
      snapshot: MonitorSnapshot,
      last_refreshed: Instant,
      dirty: bool,        // set by broadcasts arriving mid-drag
      pending_refresh: Option<Instant>, // for 200 ms debounce
  }

  fn cache() -> &'static Mutex<Cached> {
      CACHE.get_or_init(|| {
          let s = snapshot();
          Mutex::new(Cached {
              snapshot: s,
              last_refreshed: Instant::now(),
              dirty: false,
              pending_refresh: None,
          })
      })
  }

  /// Returns a fresh snapshot. Does NOT touch the cache.
  pub fn snapshot() -> MonitorSnapshot {
      let mut monitors: Vec<MonitorInfo> = Vec::with_capacity(4);
      unsafe extern "system" fn enum_proc(hmon: HMONITOR, _hdc: *mut _, _rect: *mut RECT, lp: LPARAM) -> BOOL {
          let out = &mut *(lp.0 as *mut Vec<MonitorInfo>);
          let mut mi: MONITORINFOEXW = std::mem::zeroed();
          mi.monitorInfo.cbSize = std::mem::size_of::<MONITORINFOEXW>() as u32;
          if GetMonitorInfoW(hmon, &mut mi.monitorInfo as *mut _ as *mut _).as_bool() {
              let m = &mi.monitorInfo;
              let bounds = Rect {
                  left: m.rcMonitor.left, top: m.rcMonitor.top,
                  right: m.rcMonitor.right, bottom: m.rcMonitor.bottom,
              };
              let work_area = Rect {
                  left: m.rcWork.left, top: m.rcWork.top,
                  right: m.rcWork.right, bottom: m.rcWork.bottom,
              };
              let (mut dpi_x, mut dpi_y) = (96u32, 96u32);
              let _ = GetDpiForMonitor(hmon, MDT_EFFECTIVE_DPI, &mut dpi_x, &mut dpi_y);
              let scale = ((dpi_x as f32 / 96.0) * 100.0).round() as u32;
              out.push(MonitorInfo { bounds, work_area, scale });
          }
          BOOL(1)
      }
      unsafe {
          let _ = EnumDisplayMonitors(
              None, None,
              Some(enum_proc),
              LPARAM(&mut monitors as *mut _ as isize),
          );
      }
      MonitorSnapshot { monitors }
  }

  pub fn current() -> MonitorSnapshot {
      cache().lock().unwrap().snapshot.clone()
  }
  ```

  ```rust
  const DEBOUNCE: Duration = Duration::from_millis(200);

  /// Request a cache refresh. If a drag is active (caller-signalled), mark dirty and
  /// defer. Otherwise debounce by 200 ms; if another request arrives within the window
  /// only the last one runs.
  pub fn request_refresh(drag_active: bool) {
      let mut c = cache().lock().unwrap();
      if drag_active {
          c.dirty = true;
          return;
      }
      let now = Instant::now();
      c.pending_refresh = Some(now);
      // Caller (tool_window WM_TIMER) triggers the actual refresh after DEBOUNCE.
  }

  /// Called from tool_window's timer after DEBOUNCE elapses.
  pub fn flush_pending() {
      let mut c = cache().lock().unwrap();
      let Some(t0) = c.pending_refresh else { return };
      if Instant::now().duration_since(t0) < DEBOUNCE { return; }
      c.pending_refresh = None;
      c.dirty = false;
      c.snapshot = snapshot();
      c.last_refreshed = Instant::now();
  }

  pub fn on_drag_ended() {
      let mut c = cache().lock().unwrap();
      if c.dirty {
          c.dirty = false;
          c.snapshot = snapshot();
          c.last_refreshed = Instant::now();
      }
  }
  ```

  ```rust
  use alt3rsnap::engine::geometry::Rect;
  use std::sync::{Mutex, OnceLock};
  use windows::core::{w, PCWSTR};
  use windows::Win32::Foundation::{COLORREF, HWND, LPARAM, LRESULT, WPARAM};
  use windows::Win32::Graphics::Gdi::{
      BeginPaint, CreateSolidBrush, DeleteObject, EndPaint, FillRect, FrameRect,
      GetDC, ReleaseDC, PAINTSTRUCT,
  };
  use windows::Win32::UI::WindowsAndMessaging::{
      CreateWindowExW, DefWindowProcW, RegisterClassExW,
      SetLayeredWindowAttributes, SetWindowPos, ShowWindow,
      LWA_ALPHA, SWP_NOACTIVATE, SWP_NOREDRAW, SWP_NOZORDER, SW_HIDE, SW_SHOWNA,
      WM_PAINT, WNDCLASSEXW, WS_EX_LAYERED, WS_EX_NOACTIVATE, WS_EX_TOOLWINDOW,
      WS_EX_TOPMOST, WS_EX_TRANSPARENT, WS_POPUP,
  };

  static HWND_CACHE: OnceLock<Mutex<Option<HwndBox>>> = OnceLock::new();
  static OPACITY: OnceLock<Mutex<u8>> = OnceLock::new();

  // `HWND` is not Send by default in the windows crate; wrap it.
  struct HwndBox(pub HWND);
  unsafe impl Send for HwndBox {}

  fn hwnd_slot() -> &'static Mutex<Option<HwndBox>> {
      HWND_CACHE.get_or_init(|| Mutex::new(None))
  }

  pub fn set_opacity(a: u8) {
      *OPACITY.get_or_init(|| Mutex::new(0x99)).lock().unwrap() = a;
  }

  pub fn register_class(hinstance: windows::Win32::Foundation::HINSTANCE) {
      unsafe {
          let cls = WNDCLASSEXW {
              cbSize: std::mem::size_of::<WNDCLASSEXW>() as u32,
              lpfnWndProc: Some(wndproc),
              hInstance: hinstance,
              lpszClassName: w!("Alt3rSnapOverlay"),
              ..Default::default()
          };
          RegisterClassExW(&cls);
      }
  }

  unsafe extern "system" fn wndproc(hwnd: HWND, msg: u32, wp: WPARAM, lp: LPARAM) -> LRESULT {
      match msg {
          WM_PAINT => {
              let mut ps: PAINTSTRUCT = std::mem::zeroed();
              let hdc = BeginPaint(hwnd, &mut ps);
              let fill = CreateSolidBrush(accent_color());
              FillRect(hdc, &ps.rcPaint, fill);
              DeleteObject(fill);
              let frame = CreateSolidBrush(accent_color());
              FrameRect(hdc, &ps.rcPaint, frame);
              DeleteObject(frame);
              EndPaint(hwnd, &ps);
              LRESULT(0)
          }
          _ => DefWindowProcW(hwnd, msg, wp, lp),
      }

      // Best-effort via DwmGetColorizationColor; fallback to RGB(0,120,215).
      use windows::Win32::Graphics::Dwm::DwmGetColorizationColor;
  ```

  ```rust
  use alt3rsnap::engine::state::{Action, WindowId};
  use alt3rsnap::win_api_trait::{Call, RecordingWinApi, WinApi};

  fn apply(api: &mut dyn WinApi, acts: &[Action]) {
      // Minimal local apply: mirror the adapter's per-action dispatch for the subset
      // we care about in these tests. (The real adapter function calls this plus extras;
      // the test-visible contract here is ordering.)
      for a in acts {
          match a {
              Action::ShowSnapPreview { rect } => api.overlay_show(*rect),
              Action::HideSnapPreview => api.overlay_hide(),
              Action::ApplySnapRect { hwnd, rect } => { let _ = api.set_window_rect(*hwnd, *rect); }
              Action::UpdateDrag { hwnd, new_rect } => { let _ = api.set_window_rect(*hwnd, *new_rect); }
              _ => {}
          }
      }

  fn apply_snap_rect_is_terminal_for_geometry_in_a_batch() {
      let mut api = RecordingWinApi::new();
      let r = Rect { left: 0, top: 0, right: 960, bottom: 540 };
      let batch = vec![
          Action::HideSnapPreview,

      let last_set = api.calls.iter().enumerate().rev().find(|(_, c)| matches!(c, Call::SetWindowRect(..)));
  ```
