# Alt3rSnap v0.2 M1 — Middle-click + Extensible Dispatch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship Milestone 1 of v0.2 — Alt + middle-click toggles maximize/restore on the window under the cursor, with the full click sequence swallowed, config-driven dispatch (`[behavior].middle_click_action`), tolerant string-at-file-layer parsing, and an adapter-owned swallow latch that survives stray events. Deliverable: `main` branch with all tests green and a manually-verified Windows binary.

**Architecture:** Purely additive to the v0.1 engine/adapter split. Engine gains one new event (`MiddleDown`), one new action (`ToggleMaximize`), one new enum (`MiddleClickAction`), and one new `EngineConfig` field. The adapter gains a single-purpose `SwallowLatch` module plus a `ToggleMaximize` action handler. The hook translates `WM_MBUTTONDOWN` → engine event; `WM_MBUTTONUP` is adapter-only (latch). Config gains one new `Behavior.middle_click_action` string that is parsed-and-warned in `to_engine_config()`, mirroring the existing `center_mode` pattern.

**Tech Stack:** Rust 2021, existing v0.1 crate shape — no new dependencies. New test file `tests/adapter_middle_latch.rs` runs on Linux using std primitives only (`AtomicBool` + a fake monotonic clock passed to `SwallowLatch`, no Win32).

**Milestone reference:** Spec `docs/superpowers/specs/2026-04-21-alt3rsnap-v0.2-design.md` §6.1, §2.9, §3.3, §3.5, §3.6, §4.2, §4.3, §5.2, §5.7.

- `src/swallow_latch.rs` — new adapter module, `SwallowLatch` struct with `set()`, `try_swallow()`, `on_timer()`, `on_begin_drag()`, pure-Rust (no `windows` crate), Linux-buildable.
- `tests/adapter_middle_latch.rs` — new integration test file covering the four latch cases from spec §5.7.

- `src/engine/state.rs` — add `Event::MiddleDown`, `Action::ToggleMaximize`.
- `src/engine/config.rs` — add `MiddleClickAction` enum and `EngineConfig.middle_click_action` field.
- `src/engine/mod.rs` — add `Armed + MiddleDown` handler.
- `src/config.rs` — add `Behavior.middle_click_action: String`, parse in `to_engine_config()` with warn-and-default.
- `src/hook.rs` — translate `WM_MBUTTONDOWN` → `Event::MiddleDown`; handle `WM_MBUTTONUP` via adapter latch; no engine event for mouse-up.
- `src/adapter.rs` — handle `Action::ToggleMaximize`; clear latch on any `Action::BeginDrag` before processing.
- `src/win_api.rs` — add `show_window_maximize(hwnd)` wrapper (the `is_zoomed` and `restore` helpers already exist; §3.6 of spec only requires the missing piece).
- `src/main.rs` — wire the hook to the adapter's latch-accessor (module declaration for `swallow_latch`; shared latch instance via `once_cell::sync::Lazy`-equivalent pattern using `std::sync::OnceLock`).
- `tests/engine_fsm.rs` — add the three middle-click engine cases from §5.2.
- `tests/config_parse.rs` — add tolerant-string tests for `middle_click_action`.

Files intentionally not modified: `src/engine/geometry.rs`, `src/engine/rules.rs`, `src/engine/policy.rs`, `src/engine/modifiers.rs`, `src/fullscreen.rs`, `src/tray.rs`, `src/tool_window.rs`, `src/elevate.rs`, `src/autostart.rs`, `src/dpi.rs`, `src/logging.rs`, `src/config_ops.rs`. These are scope for later milestones.

**Decomposition rationale:** `SwallowLatch` is its own module (not inlined in `adapter.rs`) because it is self-contained, testable on Linux, and will grow at most one more method (clearing on drag start) — a clean unit. The `MiddleClickAction` enum lives in `engine/config.rs` so `EngineConfig` can own it without an extra file; one enum is not enough to justify a new file.

```rust
//! Adapter-owned latch that swallows the matching WM_MBUTTONUP after the
//! engine has acted on a WM_MBUTTONDOWN (spec §2.9).
//!
//! The latch is Win32-free: it takes a monotonic-millisecond timestamp as an
//! explicit argument on every operation. The real adapter passes
//! `GetTickCount64`; tests pass a synthetic clock.

use std::sync::atomic::{AtomicBool, AtomicU64, Ordering};

/// 500 ms safety clear window (spec §2.9).
pub const SAFETY_CLEAR_MS: u64 = 500;

pub struct SwallowLatch {
    armed: AtomicBool,
    armed_at_ms: AtomicU64,
}

impl SwallowLatch {
    pub const fn new() -> Self {
        SwallowLatch {
            armed: AtomicBool::new(false),
            armed_at_ms: AtomicU64::new(0),
        }
    }

    /// Arm the latch. `now_ms` is a monotonic millisecond timestamp.
    pub fn set(&self, now_ms: u64) {
        self.armed_at_ms.store(now_ms, Ordering::Relaxed);
        self.armed.store(true, Ordering::Release);
    }

    /// True iff armed.
    pub fn is_set(&self) -> bool {
        self.armed.load(Ordering::Acquire)
    }

    /// Attempt to swallow a WM_MBUTTONUP. Returns true if the event should be
    /// dropped (the latch was armed); clears the latch in that case.
    ///
    /// `now_ms` is currently unused by this method but is taken so the
    /// signature matches the other latch operations and so a future safety
    /// check could be added here without a breaking change.
    pub fn try_swallow(&self, _now_ms: u64) -> bool {
        self.armed.swap(false, Ordering::AcqRel)
    }
}

impl Default for SwallowLatch {
    fn default() -> Self {
        Self::new()
    }
}
```

```rust
unsafe extern "system" fn mouse_hook(code: i32, wparam: WPARAM, lparam: LPARAM) -> LRESULT {
    if code < 0 {
        return CallNextHookEx(None, code, wparam, lparam);
    }
    let info = &*(lparam.0 as *const MSLLHOOKSTRUCT);
    let cursor = Point {
        x: info.pt.x,
        y: info.pt.y,

        let now = windows::Win32::System::SystemInformation::GetTickCount64();
        if crate::adapter::swallow_latch().try_swallow(now) {
            return LRESULT(1);
        }
        return CallNextHookEx(None, code, wparam, lparam);

        WM_MOUSEMOVE => Some(Event::MouseMove { cursor }),
        WM_LBUTTONDOWN => {
            let target = crate::adapter::resolve_target(cursor);
            Some(Event::LeftDown { cursor, target })
        }
        WM_LBUTTONUP => Some(Event::LeftUp),
        WM_MBUTTONDOWN => {
            let target = crate::adapter::resolve_target(cursor);
            Some(Event::MiddleDown { cursor, target })
        }
        WM_RBUTTONDOWN => {
            let target = crate::adapter::resolve_target(cursor);
            Some(Event::RightDown { cursor, target })
        }
        WM_RBUTTONUP => Some(Event::RightUp),
        _ => None,
    };

    let swallow = if let Some(ev) = event {
        let actions = ENGINE.with(|e| e.borrow_mut().handle(ev));
        crate::adapter::apply_actions(&actions)
    } else {
        false
    };

    if swallow {
        LRESULT(1)
    } else {
        CallNextHookEx(None, code, wparam, lparam)
    }
}
```
