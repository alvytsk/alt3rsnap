# v0.2.1 Polish & Release Automation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bump Alt3rSnap to v0.2.1 with a clean lint config, trimmed dependencies, and a tag-triggered GitHub Actions workflow that publishes a UPX-packed `.exe` + SHA-256 to GitHub Releases.

**Architecture:** Five independent, sequential tasks — version bookkeeping → `rustfmt.toml` + reformat → `[lints]` in manifest → unused `windows` feature removal → release CI workflow. No engine or adapter logic changes anywhere.

**Tech Stack:** Rust (stable), `cargo fmt`, `cargo clippy`, `cargo check`, GitHub Actions (`windows-latest`), UPX (via Chocolatey), `softprops/action-gh-release@v2`.

## Task 5: Release CI workflow

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Create the release workflow**

Create `.github/workflows/release.yml`:

```yaml
name: Release

on:
  push:
    tags: ['v*.*.*']

jobs:
  release:
    runs-on: windows-latest
    permissions:
      contents: write

    steps:
      - uses: actions/checkout@v4

      - uses: dtolnay/rust-toolchain@stable

      - uses: Swatinem/rust-cache@v2

      - name: Build release binary
        run: cargo build --release --target x86_64-pc-windows-msvc

      - name: Install UPX
        run: choco install upx -y

      - name: Pack binary with UPX
        run: upx --best --lzma target/x86_64-pc-windows-msvc/release/alt3rsnap.exe

      - name: Compute SHA-256
        run: |
          $hash = (Get-FileHash target/x86_64-pc-windows-msvc/release/alt3rsnap.exe -Algorithm SHA256).Hash.ToLower()
          "$hash  alt3rsnap.exe" | Out-File -Encoding ascii alt3rsnap-sha256.txt
          Get-Content alt3rsnap-sha256.txt

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: |
            target/x86_64-pc-windows-msvc/release/alt3rsnap.exe
            alt3rsnap-sha256.txt
```

- [ ] **Step 2: Verify workflow YAML is valid**

```bash
python3 -c "import yaml; yaml.safe_load(open('.github/workflows/release.yml'))" && echo "YAML OK"
```

Expected: `YAML OK`. If Python isn't available, use `cat .github/workflows/release.yml` and visually inspect indentation — YAML whitespace errors are the most common failure mode.

- [ ] **Step 3: Confirm ci.yml is unchanged**

```bash

```

Verify it still has the same `on: push: branches: [main]` trigger and both jobs (`lint-and-test`, `windows-build`). The release workflow must not have overwritten it.

- [ ] **Step 4: Commit**

```bash

```

If the release job fails, check: (a) UPX choco install output, (b) whether `softprops/action-gh-release` needs a `token:` key (it uses `GITHUB_TOKEN` automatically — no secret needed), (c) whether the binary path is correct.
